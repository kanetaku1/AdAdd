# バックエンドAPI接続およびCORS設定に関するガイド

このドキュメントは、AdAddのフロントエンド開発チーム向けに、完成したバックエンドAPIへの接続方法と関連する機能の活用についてまとめたものです。

## 1. バックエンドとの通信（CORS）について
バックエンドのAPIサーバー（`http://localhost:8080`）側にて、フロントエンド（`http://localhost:3000`）からのクロスオリジン通信を許可する **CORS（Cross-Origin Resource Sharing）設定の実装が完了** しました。
これにより、ブラウザのセキュリティ制限（blocked by CORS policy等のエラー）を受けることなく安全にAPI通信が可能です。

- **許可済みのOrigin (ローカル開発環境):** `http://localhost:3000`, `http://localhost:3001`
- **許可済みのヘッダー:** `Origin`, `Content-Type`, `Accept`, `X-User-ID`, `X-User-Roles`, `Authorization`
- **許可済みのメソッド:** `GET`, `PUT`, `POST`, `DELETE`, `OPTIONS`, `PATCH`

## 2. API通信モードへの切り替え手順
フロントエンドの開発コード（`src/lib/api/client.ts` など）には、環境変数の有無に応じて「API・MySQL通信モード」と「モック（ダミー）データモード」を自動で切り替える非常に優秀な機能がすでに実装されています。
本番のバックエンドAPIを利用してデータベースと連携するためには、以下の手順で設定を行ってください。

1. `apps/web/` フォルダの中に、雛形である `.env.example` をコピーして **`.env` ファイルを作成**してください。
2. 作成した `.env` の中に以下の一文が設定されていることを確認します。
   ```env
   NEXT_PUBLIC_API_BASE_URL=http://localhost:8080
   ```
3. 設定後、ターミナルで実行中の Next.js サーバー（`npm run dev`）を一度停止し、**再起動**してください。
   これにより、フロントエンド全体が自動的に「API通信モード」に切り替わります。

## 3. UI側（画面側のコード）に関する重要なお願い
「API通信モード」の自動切り替え機能は素晴らしい設計ですが、現在**一部の画面のソースコード**において、この切り替え機能の処理を通さず、**モックデータを直接importして表示する古い仮組みのコードが残ったまま（未適応）** になっています。

そのため、`.env` でAPI通信モードをONにしても、以下の画面などではデータがバックエンドに連携されず、ダミーデータが出力され続けてしまいます。

**【対応が残っている主な箇所】**
- **ダッシュボード画面 (`src/app/page.tsx`)**
- **企業一覧画面 (`src/app/companies/page.tsx`)** など

これらの画面のソースコード冒頭を見ると、`import { mockCompanies } from "@/lib/mock/companies"` といった形でダミーデータを直接読み込んでいます。
お手数ですが、すでに他の画面（協賛詳細など）で実装していただいたのと同様に、これら未対応の画面についても**「自動切り替え機能 (`isApiEnabled()` など)」を経由してAPIからデータを取得する（またはダミーを返す）コードへの繋ぎ変え**をお願いいたします。

以上の対応が完了すれば、現状のモック状態から脱却し、バックエンドのデータベースと完全に連動した本番システムとしての動作が実現します。
